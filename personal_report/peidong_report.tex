\section{Technical Understanding}

\subsection*{2a. Technical Challenge: Context Window Optimization (A1d)}

A significant challenge was that frozen BioBERT embeddings (A1d) initially underperformed sparse baselines. I diagnosed this as a \textbf{context window limitation}: the default sentence-transformer configuration truncated inputs at 100 tokens, losing critical evidence. I re-implemented the extraction pipeline using \texttt{AutoModel} with a 384-token window and mean-pooling. This optimization raised test macro-F1 from 0.353 to 0.419 (+18.7\%), demonstrating that representation quality was limited by encoding parameters rather than model expressiveness. Additionally, I implemented weighted cross-entropy to handle the extreme class imbalance (11\% \emph{maybe} class, all results are recorded in \texttt{AITA\_Biobert\_fit\_on\_q\_ctx.ipynb}).

\subsection*{2b. Impact on Group Results}

My Axis-1 experiments provided the team's central empirical finding: while sparse models plateau regardless of input richness, transformer models can leverage extended context (A1f), achieving 0.466 macro-F1. This established the "representation matters" narrative and motivated our final transition to PubMedBERT (A1g), which yielded our peak result of 0.604. My work ensured that the baseline and intermediate transformer experiments were statistically robust and properly weighted.

\subsection*{2c. Reflection and Future Work}

Exploratory zero-shot testing with the Gemini API achieved 0.611 macro-F1, surpassing our fine-tuned models. This indicates that future work should explore generative reasoning (e.g., Chain-of-Thought) to improve evidence detection for the \emph{maybe} class, moving beyond the representation-focused scope of Axis 1.
